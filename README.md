+# 物品貸出申請フォーム（Google Apps Script）
+
+大学の総務部向けに、紙申請を電子化するための Web アプリ（GAS）です。
+
+## 実装内容
+
+- フォーム上に物品画像一覧（Box）リンクを掲載
+  - https://kanagawa-u.box.com/s/gp9fhj9hrlbysisik7tm77379xx8x5ho
+- 申請フォーム項目
+  - 申請部署
+  - 申請者氏名
+  - メールアドレス
+  - Tel
+  - 借用開始日
+  - 受取時間
+  - 返却日
+  - 返却時間（受取/返却時間はプルダウンで10分単位: 00/10/20/30/40/50）
+  - 借用物品（複数選択可、数量は各物品の初期在庫を上限にプルダウン表示）
+- 物品マスタ管理画面
+  - 物品の追加・削除
+  - 初期在庫（台帳の残数計算に使用）
+  - 有効/無効
+- 台帳日付管理
+  - 開始日/終了日を指定して日別台帳の日付・曜日を作成
+  - 同時に申請シートの内容から貸出数/残数を再反映
+- 採番
+  - 申請No: `YYYY + 5桁連番`
+  - 年度は 4 月開始（例: 2026/4〜2027/3 は 2026年度）
+- 申請データ保存
+  - 1申請で複数物品を選択した場合、物品ごとに1行で保存
+  - 同じ申請Noで通番（1,2,3...）を付与
+  - 申請ボタン押下後は「申請中です。しばらくお待ちください・・・」を表示
+  - 申請完了後は確認画面へ遷移し、申請内容を表示
+  - 確認画面から「続けて申請する」で申請画面へ戻れる
+  - 入力メールアドレスへ申請内容を自動送信
+- 台帳連携
+  - 申請確定時に日別台帳の貸出数/残数を更新
+  - 残数がマイナスになったセルは赤色でアラート表示
+- モダンUI / 書式
+  - フォーム/管理画面をカード型・グラデーション基調のUIに調整
+  - スプレッドシートはヘッダー色・フォント・日付書式を自動整形
+
+## ファイル構成
+
+- `Code.gs`: サーバー側ロジック（検証、採番、台帳更新、マスタ管理、シート書式整形）
+- `Index.html`: 申請フォーム
+- `Admin.html`: 物品マスタ管理画面
+- `appsscript.json`: GAS マニフェスト

+## 初期セットアップ
+
+1. GAS プロジェクトにこのファイル群を配置
+2. Script Properties に以下を設定
+   - `APPLICATION_SHEET_ID`: 申請管理用スプレッドシート ID
+   - `LEDGER_FISCAL_YEAR`（任意）: 日別台帳を事前作成する年度（例: `2026`）
+
+### Script Properties 設定の詳しい手順
+
+
+### Script Properties（スクリプトプロパティ）を小学生向けに説明
+
+- **Script Properties** は、アプリに教える「ひみつのメモ帳」です。
+- 入力欄が **「キー」** ではなく **「プロパティ」** と表示されても、意味は同じです。
+  - **プロパティ名（= キー）**: メモの「タイトル」
+  - **値**: メモの「中身」
+- このアプリでは、次のメモが必要です。
+  1. プロパティ: `APPLICATION_SHEET_ID` / 値: 申請管理シートのID
+
+#### 迷わない入力見本（そのまま真似OK）
+
+1. Apps Script を開く
+2. 左の⚙（設定）を押す
+3. 「スクリプト プロパティ」で「追加」を押す
+4. 1行目を次のように入れる
+   - **プロパティ**: `APPLICATION_SHEET_ID`
+   - **値**: 申請管理シートのID（長い英数字）
+5. 保存ボタンを押す
+
+#### 「ID（長い英数字）」ってどこ？
+
+GoogleスプレッドシートのURLの、`/d/` と `/edit` の間です。
+
+- 例: `https://docs.google.com/spreadsheets/d/ABC123xyz456DEF/edit#gid=0`
+- この場合のIDは **`ABC123xyz456DEF`**
+
+#### うまくいったか確認する方法
+
+- Apps Script で `initializeSheets()` を実行
+- エラーが出なければOK
+- もしエラーが出たら、次をチェック
+  - プロパティ名のつづり（大文字小文字）
+  - 値に余計な空白が入っていないか
+  - IDが別のシートのものになっていないか
+
+#### 1) スプレッドシート ID を控える
+
+1. 申請管理用スプレッドシートをブラウザで開く
+2. URL の `/d/` と `/edit` の間が ID
+   - 例: `https://docs.google.com/spreadsheets/d/xxxxxxxxxxxxxxxxxxxx/edit#gid=0`
+   - この `xxxxxxxxxxxxxxxxxxxx` が `APPLICATION_SHEET_ID`
+3. 日別台帳は同じスプレッドシート内の別シートとして自動作成されます。
+
+#### 2) Apps Script エディタで Script Properties を開く
+
+1. 該当の Apps Script プロジェクトを開く
+2. 左メニューの **プロジェクトの設定（歯車アイコン）** をクリック
+3. 画面下部の **スクリプト プロパティ** セクションで **スクリプト プロパティを追加** をクリック
+
+#### 3) キーと値を登録する
+
+以下を追加して保存:
+
+| プロパティ（= キー） | 値 |
+|---|---|
+| `APPLICATION_SHEET_ID` | 申請管理用スプレッドシートID |
+
+> キー名は `Code.gs` で固定参照しているため、スペルを完全一致させてください。
+
+#### 4) 設定確認（推奨）
+
+`initializeSheets()` を実行して、以下が作成されることを確認します。
+
+- 申請管理側: `申請`, `物品マスタ`, `採番管理`
+- 日別台帳: 指定年度（4月〜翌3月）の日付・曜日行を自動作成
+
+キー名やIDが未設定の場合は、実行時に次のエラーが出ます。
+
+- `Script Properties に APPLICATION_SHEET_ID を設定してください。`
+
+#### 5) 権限の注意点
+
+- Webアプリ実行ユーザー（`executeAs: USER_DEPLOYING`）に、申請管理スプレッドシート（同一ブック内の日別台帳を含む）への編集権限が必要です。
+- スプレッドシートが別組織/共有ドライブにある場合は、実行ユーザーがアクセスできるかを事前に確認してください。
+初期セットアップの続き:
+- エディタで `initializeSheets()` を実行（必要なシート/ヘッダ作成）
+- Web アプリとしてデプロイ
+  - フォーム: `.../exec?page=form`（既定）
+  - マスタ管理: `.../exec?page=admin`

+### 管理画面で「保存」ボタンを押す場所（迷ったとき用）
+
+「保存」ボタンは、**物品マスタの表のいちばん下**にあります。
+
+1. `.../exec?page=admin` を開く
+2. 物品マスタの表に、物品コード・物品名・在庫などを入力

+#### 保存ボタンが見えないとき
+
+- 画面表示を 100% に戻す（拡大しすぎると下のボタンが見えにくいです）
+- ブラウザのズームを下げる（`Ctrl` + `-`）
+- 一番下までスクロールする
+- スマホの場合は横向きにして表示領域を広げる
+

+### 日別台帳の事前作成ルール（年度4月〜翌3月）
+
+- `initializeSheets()` 実行時に、`日別台帳` シートへ対象年度の **4/1〜翌3/31** を自動作成します。
+- 日別台帳は次の形式です。
+  - A列: 日付
+  - B列: 曜日
+  - C列以降: 各物品の「貸出数」「残数」列
+- 対象年度は次の優先順で決まります。
+  1. Script Properties の `LEDGER_FISCAL_YEAR`（例: `2026`）
+  2. 未設定時は実行日の年度（4月始まり）
+
+### 在庫超過のアラート表示
+
+- フォーム側の数量プルダウンは、各物品の初期在庫を上限に表示します。
+- `日別台帳` では、貸出により残数がマイナスになったセルを赤色表示します。
+- 赤セルを見ることで、どの日付・どの物品で超過が発生したかを確認できます。
+
+## 物品マスタの内容をフォームの「借用物品」に反映する方法
+
+結論: **管理画面（`.../exec?page=admin`）で保存すると、次回フォーム読込時に自動反映**されます。
+
+### 反映の流れ（かんたん）
+
+1. 管理画面（`.../exec?page=admin`）を開く
+2. 物品コード / 物品名 / 初期在庫 / 有効チェックを入力・修正
+3. **保存** を押す（物品マスタのみ反映）
+4. 必要に応じて「台帳日付管理」で開始日/終了日を指定して **日付を台帳に反映** を押す
+5. 申請フォーム（`.../exec?page=form`）を開く
+6. 借用開始日・返却日を入れて **「借用物品を再読込」** を押す
+7. 最新の物品マスタが借用物品一覧に表示される
+
+### どの項目がどう反映される？
+
+- `物品名` → フォームの物品名表示
+- 数量プルダウンは各物品の初期在庫を上限に表示
+- `有効` が OFF（FALSE） → フォームに表示しない
+
+### 反映されないときのチェック
+
+- 管理画面で **保存** できているか
+- 物品コードと物品名が空欄になっていないか（空欄行は保存対象外）
+- フォーム側で **「借用物品を再読込」** を押したか
+- `APPLICATION_SHEET_ID` が正しい申請管理シートを向いているか
+
+## スプレッドシート仕様（1つのスプレッドシート内に集約）
+
+### 基本シート
+
+- `申請`
+  - 登録日時, 申請No, 通番, 申請部署, 申請者氏名, メールアドレス, Tel,
+    借用開始日, 受取時間, 返却日, 返却時間, 物品コード, 物品名, 数量
+- `物品マスタ`
+  - 物品コード, 物品名, 初期在庫, 有効
+- `採番管理`
+  - 年度, 最新連番
+
+### 日別台帳（同一スプレッドシート内の別シート）
+
+- `日別台帳`
+  - 1行=1日付
+  - 列は「日付 + 各物品の貸出数/残数」が横方向に並ぶ
+  - 例: `日付, 物品A_貸出数, 物品A_残数, 物品B_貸出数, 物品B_残数`
